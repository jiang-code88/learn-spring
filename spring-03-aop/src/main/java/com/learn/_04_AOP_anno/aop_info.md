0 AOP 的意义
- 弥补 OOP 设计模式不能完全有效的解决这些分散的相同逻辑提取问题。
- 核心的作用是「解耦」

1 AOP 术语
- Target 目标对象
- Proxy 代理对象
- JoinPoint 连接点（目标对象类中定义的所有方法）
- Pointcut 切入点（切入点可以是 0 个或多个（甚至全部）连接点的组合）
- Advice 通知（在连接点的切入点上增强的逻辑代码）
- Weaving 织入（将 Advice 通知应用到 Target 目标对象，进而生成 Proxy 代理对象的过程）

  Proxy 代理对象 = Target 目标对象 + Advice 通知
  Aspect 切面 = PointCut 切入点 + Advice 通知
  连接点代表可以切入的方法，切入点代表实际进行切入的连接点。

2 通知类型
- Before 前置通知：目标对象的方法调用之前触发
- After 后置通知：目标对象的方法调用之后触发
- AfterReturning 返回通知：目标对象的方法调用完成，在返回结果值之后触发
- AfterThrowing 异常通知：目标对象的方法运行中抛出 / 触发异常后触发
  AfterReturning 与 AfterThrowing 两者互斥
  如果方法调用成功无异常，则会有返回值；如果方法抛出了异常，则不会有返回值。
- Around 环绕通知：编程式控制目标对象的方法调用，可以直接拿到目标对象，以及要执行的方法。
  环绕通知可以任意的在目标对象的方法调用前后搞事，甚至可以不调用目标对象的方法。
  编写 InvocationHandler 或 MethodInterceptor 的匿名内部类就是实现的环绕通知。

3 切入点表达式
execution(
    public：限定只切入 public 类型的方法（访问修饰符不写的话，表示切入所有访问修饰符的方法）
    void：限定只切入返回类型为 void 的方法
    com.learn._04_AOP_anno.service.FinanceService.：限定切入方法所在类的全限定名
    subtractMoney：限定切入的方法名
    (double)：限定切入方法的参数列表类型（对于基本数据类型，直接声明即可 / 引用数据类型则要写类的全限定名）
)

4 切入点表达式通配符
- 通配符 `*` 表示不限制：
  可用于返回类型，表示不限制返回类型，返回什么的都可以切入，
  可用于方法名，表示不限定方法名，什么方法都可以切入。
- 如果使用通配符 `*` 覆盖到接口类，则这个接口的所有实现类上的匹配方法都会被切入增强。
- 如果使用通配符 `*` 在方法参数上，表示切入任意类型的单个参数方法，不能匹配没有参数或多个参数。 
  使用通配符 `..` 在方法参数上，表示匹配不限参数类型和参数个数（可以没有参数或多个参数）。
- 如果通配符使用在包名中，`*` 表示单个包，`..` 表示多个包。

5 抛出异常的方法切入
- 如果方法参数后有用 throws 抛出异常的，切入表达式可以写上 throws 和抛出的类型的全限定名
